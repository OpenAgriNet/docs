# Glossary

OpenAgriNet spans three knowledge domains: Indian agriculture policy, the Beckn open protocol, and the platform's own components. This glossary covers all three so that readers from any background can navigate the system.

## Agricultural & Policy Terms

::: info
These terms reflect current deployments, which are India-focused. Future regional variants will add their own terminology.
:::

| Term | Definition |
|------|-----------|
| **Mandi** | Agricultural market/marketplace where farmers sell produce. Regulated by APMC in each state. |
| **APMC** | Agricultural Produce Market Committee — state-level body that regulates mandis and sets trading rules. |
| **MSP** | Minimum Support Price — government-guaranteed floor price for key crops, intended to protect farmers from market volatility. |
| **PMFBY** | Pradhan Mantri Fasal Bima Yojana — India's national crop insurance scheme providing coverage against natural calamities, pests, and diseases. |
| **Kharif** | Summer/monsoon cropping season (June–October). Major crops: rice, maize, cotton, soybean. |
| **Rabi** | Winter cropping season (October–March). Major crops: wheat, barley, mustard, chickpea. |
| **IMD** | India Meteorological Department — primary source of weather data and forecasts for agricultural planning. |
| **AgriStack** | India's federated digital infrastructure for agriculture — a set of databases and APIs providing farmer, land, and crop information. |
| **MahaDBT** | Maharashtra Direct Benefit Transfer portal — state government platform for scheme applications and subsidy disbursement. |
| **FPO** | Farmer Producer Organization — a collective of farmers organized as a company or cooperative for better market access and bargaining power. |
| **KVK** | Krishi Vigyan Kendra (Farm Science Centre) — district-level agricultural extension centers providing training and advisory to farmers. |
| **CHC** | Custom Hiring Centre — facilities where farmers can rent agricultural equipment (tractors, harvesters, sprayers) they can't afford to own. |

## Beckn Protocol & Network Terms

| Term | Definition | System Mapping |
|------|-----------|----------------|
| **Beckn** | Open protocol for decentralized commerce and services networks. Enables any provider to be discoverable by any consumer without bilateral integration. | Core interoperability protocol used across all OAN deployments |
| **BAP** | Beckn Application Platform — the consumer/aggregator side of the network. Sends search requests on behalf of users. | Seeker Service acts as BAP |
| **BPP** | Beckn Provider Platform — the provider side of the network. Responds to search requests with catalogs of available services. | Provider Service + AI APIs act as BPPs |
| **DSEP** | Distributed Search & Enquiry Protocol — a Beckn specification designed for discovery and enquiry (not commerce). Fits agriculture's information-discovery pattern. | Used for catalog search/select across all domains |
| **Context** | Request metadata in every Beckn message: domain, action, timestamp, TTL, BAP/BPP identifiers. | `context` field in all API requests; `context.domain` drives routing |
| **Intent** | The user's search query structure within a Beckn message. Contains descriptors, item filters, and fulfillment preferences. | `message.intent` in search requests |
| **Catalog** | A provider's structured offering — the list of items, categories, and fulfillments available. Returned in response to search requests. | Generated by `SchemeCatalogGenerator`, `WeatherCatalogGenerator`, etc. |
| **Descriptor** | A name + description object used throughout Beckn messages to describe entities (providers, items, categories). | Used in items, providers, categories across all entities |
| **Fulfillment** | The delivery or service method for an item (e.g., Online, Offline, In-Person). | `fulfillment.type` field in provider responses |
| **DPI** | Digital Public Infrastructure — open, interoperable technology platforms that operate as public goods (like UPI for payments, Aadhaar for identity). | OpenAgriNet's foundational design philosophy |
| **Open Network** | An interoperable network of BAPs and BPPs connected via the Beckn protocol. No single entity controls the network. | Multiple independent OAN deployments connected via Beckn |

## Platform-Specific Terms

| Term | Definition | Component |
|------|-----------|-----------|
| **OAN** | Open AgriNet — the overall platform and coalition building interoperable agricultural information infrastructure. | Root project |
| **VISTAAR** | "Extension" or "expansion" — India's agricultural extension network model for delivering information and services to farmers at scale. | National deployment concept |
| **MahaVistaar** | Maharashtra state deployment of OpenAgriNet, focused on state-specific schemes, agricultural services, and Marathi language support. | `mh-oan-api` service |
| **Bharat OAN** | National India deployment focused on central government schemes and Hindi language support. | `bharat-oan-api` service |
| **AMUL OAN** | AMUL (Anand Milk Union Limited) dairy cooperative deployment, focused on dairy farming, veterinary advisory, and Gujarati/Marathi support. | `amul-oan-api-check` service |
| **Provider** | An entity offering agricultural services or information through the network. Can be a government department, a weather service, a market data aggregator, etc. | `OAN-Provider-Service` (NestJS) |
| **Seeker** | An end-user or application searching for agricultural services through the network. | `OAN-Seeker-Service` (NestJS) |
| **Catalog Generator** | A component that transforms raw data from a source (database, API, file) into a Beckn-compliant catalog structure. | `SchemeCatalogGenerator`, `WeatherCatalogGenerator`, `WarehouseCatalogGenerator` |
| **Item** | A specific service or product within a provider's catalog (e.g., a weather forecast, a government scheme, a market price report). | `item.entity.ts` in Provider Service |
| **Tag / TagValue** | Metadata attributes attached to items for filtering and display. Tags define attribute types; TagValues hold specific values. | `tag.entity.ts`, `tag-value.entity.ts` in Provider Service |
